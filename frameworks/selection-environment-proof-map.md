# Selection Environment Proof Map

## Core Idea

Cybersecurity hiring is not competence-based by default. It is selection-environment-based.

Different layers select for different signals.

## Selection Layers

| Layer | What it selects for | Candidate mistake | Better signal |
|---|---|---|---|
| ATS / HR | Keywords, titles, authorization, salary, obvious fit | Assuming skill will be inferred | Resume alignment, role keywords, clean titles, clear eligibility |
| Recruiter | Fit, risk reduction, communication, presentability | Overloading technical detail | Short proof summary, clear story, confidence without overclaiming |
| Hiring Manager | Operational usefulness, ramp speed, workload reduction | Talking in concepts | Role-specific project proof and workflow evidence |
| Technical Reviewer | Evidence, judgment, tool fluency, troubleshooting | Showing only credential logos | Investigation notes, detections, playbooks, queries, false-positive reasoning |
| Team / Client | Trust, documentation, escalation, communication | Treating security as solo puzzle solving | Clear handoffs, timelines, decision notes, client-safe language |

## Operating Rule

Do not merely become more competent. Make competence inspectable.

## Artifact Test

A strong artifact should answer at least three of these:

```text
[ ] What scenario is being investigated?
[ ] What evidence was collected?
[ ] What decision was made?
[ ] What uncertainty remains?
[ ] What false positives are possible?
[ ] What framework or technique does this map to?
[ ] What action should an analyst take next?
[ ] What would improve the detection, process, or control?
```
